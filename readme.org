* Overview

This tutorial guides you through installing NixOS Mobile on a OnePlus 6T (fajita) device. The process involves building a minimal base image using the local.nix file (with fixes to make it build on x86 machines), flashing it to your device, then rebuilding on the phone itself (aarch64) with the full configuration.nix and flake method we're all used to in the NixOS ecosystem.

video tutorial; https://youtu.be/yxfDNqZ9WTM

⚠️ **Security**: The configuration in this repo enables SSH with
password authentication and root login, and ships with a default
password (=nixtheplanet=), purely to make the first boot possible.
**Change =users.users.root.password= and
=users.users.user.initialPassword= in configuration.nix before
connecting the phone to any network you don't fully trust**, or
switch to SSH key authentication and disable password auth once
you're set up.

* Clone Your Configuration Repository

First, clone or download this phoneputer repository to get you started with a minimal config to build:

#+begin_src sh
git clone https://github.com/rinnvxv/phoneputer.git
cd phoneputer
#+end_src

* Build Flashable Images

We will import the official mobile-nixos repo to build from. It will be a folder `mobile-nixos` under our phoneputer project directory. After building, we can delete it as it is only needed to build our base image for one-time use:

#+begin_src sh
git clone https://github.com/mobile-nixos/mobile-nixos.git
cd mobile-nixos
cp ../local.nix .
nix-build --argstr device oneplus-fajita -A outputs.android-fastboot-images --impure
#+end_src

* Prepare the Phone

** Update OxygenOS
1. Update OxygenOS to the latest official version
2. Remove all Google accounts to avoid FRP (Factory Reset Protection) blocks

** Enable Developer Options
1. Go to Settings → About Phone
2. Tap "Build Number" 7 times
3. Go back to Settings → Developer Options
4. Enable "USB Debugging"
5. Enable "OEM Unlocking"

** Unlock the Bootloader

Open a nix-shell with Android tools:

#+begin_src sh
nix-shell -p adb-sync android-tools
#+end_src

Check device connection:

#+begin_src sh
adb devices
#+end_src

Reboot to bootloader:

#+begin_src sh
adb -d reboot bootloader
#+end_src

Unlock bootloader (this will erase all data!):

#+begin_src sh
fastboot oem unlock
#+end_src

⚠️ **Warning**: Unlocking the bootloader will erase all data on your device!

** Copy Partitions (A/B Safety Trick)

In some cases, the inactive slot can be unpopulated or contain much older firmware than the active slot, leading to various issues including a potential hard-brick. We can ensure none of that will happen by copying the contents of the active slot to the inactive slot.

To do this, sideload the copy-partitions-20220613-signed.zip package:

1. On the device, select "Apply Update", then "Apply from ADB" to begin sideload
2. On the host machine, download and sideload the package:

#+begin_src sh
wget https://mirrorbits.lineageos.org/tools/copy-partitions-20220613-signed.zip
adb -d sideload copy-partitions-20220613-signed.zip
#+end_src

Now reboot to recovery by tapping "Advanced", then "Reboot to recovery".

** Prepare for Mainline

This assumes the phone will only run SDM845 mainline images. You will need to erase the dtbo partitions for both slots:

#+begin_src sh
fastboot erase dtbo_a
#+end_src

#+begin_src sh
fastboot erase dtbo_b
#+end_src

* Flash Images to Device

Flash the critical partitions:

#+begin_src sh
./result/flash-critical.sh
#+end_src

Erase and reformat userdata, then flash the system image. This step
is what actually gets overwritten every time you rebuild the base
image, so re-run it whenever you change =local.nix= and rebuild:

#+begin_src sh
fastboot erase userdata
fastboot format:ext4 userdata
fastboot flash userdata system.img
#+end_src

If =system.img= is larger than the =system= partition can hold on
this device, flash it to =userdata= instead as shown above (fajita
uses userdata as the main rootfs partition with this build).

⚠️ **Note**: If the phone isn't recognized, try connecting it *after* booting into fastboot mode.

* First Boot and Configuration

** SSH into the Phone

NixOS Mobile provides SSH access because we set it up in the local.nix base configuration we've built. Connect to the device:

#+begin_src sh
ssh root@172.16.42.1
#+end_src

If you don't know the phone's address, check the USB interface that
appeared on your computer when you plugged the phone in (=ip a= on
Linux) — the phone is almost always reachable at =172.16.42.1= over
that link with this device's default gadget networking.

** Connect network via Computer

Rebuilding with the full flake requires internet access on the phone.
Since it's only connected via USB, we share the PC's connection over the USB link (NAT).

On PC (Change wlp0s20fs and enp0s20f0u3 to your port)
#+begin_src sh
ip a
sudo sysctl -w net.ipv4.ip_forward=1
sudo iptables -t nat -A POSTROUTING -o wlp0s20f3 -j MASQUERADE
sudo iptables -A FORWARD -i enp0s20f0u3 -o wlp0s20f3 -j ACCEPT
sudo iptables -A FORWARD -i wlp0s20f3 -o enp0s20f0u3 -m state --state RELATED,ESTABLISHED -j ACCEPT
#+end_src

On Phone
#+begin_src sh
ip route add default via 172.16.42.2
ping -c 3 8.8.8.8
#+end_src

If resolv.conf isn't blank
#+begin_src sh
cat /etc/resolv.conf
echo "nameserver 8.8.8.8" > /etc/resolv.conf
#+end_src

Note: these NAT rules and the manual =ip route=/=resolv.conf= fixes
are only needed for this *first* rebuild, while you're still running
the minimal =local.nix= image. The =configuration.nix= in this repo
already includes =networking.networkmanager.insertNameservers= and a
dnsmasq server on the phone's USB interface, so once you've
successfully switched to it, your computer will get an IP
automatically over USB and DNS will keep working across reboots —
you shouldn't need to repeat this section afterwards.

** Clone Your Configuration Repository

Once connected to the device we will rebuild without using the local.nix but like we're used to on a normal NixOS flake install. Adjust the configuration.nix to your liking, but I would suggest building the included one first. It will attempt to boot you into GNOME (see "Known Issues" below if the desktop doesn't come up):

#+begin_src sh
git clone https://github.com/rinnvxv/phoneputer.git
cd phoneputer
#+end_src

** Rebuild with Your Flake Configuration

If the phone's USB interface uses a larger MTU than the shared path can handle, large downloads/builds will hang partway through.
Lowering MTU on both ends fixes this:

On PC
#+begin_src sh
sudo ip link set enp0s20f0u3 mtu 1400
#+end_src

On Phone
#+begin_src sh
ip link set usb0 mtu 1400
nixos-rebuild switch --flake .#phoneputer
#+end_src

** Set Up a Normal User (recommended)

Working as =root= over SSH is fine to get started, but for day to
day use it's worth switching to the =user= account defined in
configuration.nix:

#+begin_src sh
ssh user@172.16.42.1
sudo nixos-rebuild switch --flake .#phoneputer
#+end_src

If you'd rather not type your password for every =sudo= command,
add this to configuration.nix (understand that this weakens local
security in exchange for convenience):

#+begin_src nix
security.sudo.wheelNeedsPassword = false;
#+end_src

If you previously cloned this repo as =root= (e.g. into
=/root/phoneputer=), move it into your normal user's home directory
so you can edit and rebuild without needing root for everything:

#+begin_src sh
sudo mv /root/phoneputer /home/user/phoneputer
sudo chown -R user:users /home/user/phoneputer
#+end_src

* What's Next?

After the rebuild completes, your OnePlus 6T will have:
- GNOME desktop environment (with bloat removed) — see Known Issues if it doesn't render
- SSH access for remote development
- Your custom package selection
- Full NixOS declarative configuration

You can now:
- Customize your configuration.nix further
- Add more packages or services
- Rebuild anytime with `nixos-rebuild switch --flake .#phoneputer`
- Use the device as a full Linux development environment

* 4G / LTE Support

Once you have the base install working, check out the [[https://github.com/mwlaboratories/phoneputer/tree/4g-testing][4g-testing branch]] for a follow-up tutorial that gets cellular connectivity working. It uses a [[https://github.com/mwlaboratories/mobile-nixos/tree/sdm845-bleeding-edge][fork of mobile-nixos]] with an upgraded kernel (6.19) and critical module fixes. 4G, frontal camera, and more are all functional.

Note: this branch targets OnePlus 6 (enchilada); adapting it to
fajita follows the same pattern as this tutorial (swap the device
string, expect to re-verify boot stability on your own).
